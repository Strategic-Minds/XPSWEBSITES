# Legacy XPSWEBSITES V2 Upgrade Directive

## Purpose
`XPSWEBSITES` is the legacy V1 reference system. It is no longer the final production target. It must be treated as a visual, content, and workflow reference that requires auto-fixing, hardening, and alignment to a stronger V2 operating model.

## Non-negotiable rules
- Do not reinvent the approved branding language.
- Do not abandon the clean white background plus alternating black sections.
- Do not downgrade the premium Phoenix Epoxy Pros look and feel.
- Use this repo as a legacy inspiration source only.
- If V1 conflicts with the approved V2 mockups, V2 workflow docs, or canonical Drive package, V2 wins.

## What must be auto-fixed from V1
- inconsistent workflow completion
- missing or partial dashboards
- weak lead-form to estimator to dashboard handoff
- incomplete automation coverage
- incomplete admin operations tooling
- incomplete visualizer architecture
- incomplete builder documentation
- incomplete validation and self-improvement loop

## Required V2 alignment goal
The final V2 system must align with the strongest workflow patterns available in elite field-service and customer-tracking systems for this business class.

## What to harvest from V1
- branding cues
- color chart references
- wording patterns worth keeping
- reusable working UI fragments
- any operational ideas that still strengthen V2

## What not to preserve blindly
Broken patterns, half-finished systems, generic sections, disconnected dashboard ideas, or anything that lowers conversion, clarity, confidence, or enterprise readiness.

## Success condition
V1 becomes a disciplined reference source. Canonical truth for the future build lives in the locked V2 Drive package, approved mockups, canonical manifests, and production build specs.
